Workflow Overview
Workflows in caNanoLab must be incorporated into the system by the caNanoLab team; they become workflow templates. caNanoLab 1.2 has implemented one workflow, a reference workflow contributed by the Nanotechnology Characterization Laboratory (NCL). Assays and assay types in this workflow are predefined based on NCL requirements. Any Researcher with assigned permissions can use the workflow template in caNanoLab to load assay input files, assign aliquots, and load output files (assay results).
Working in a workflow consists of creating Assay Runs with associated aliquots and input and output files. You can also add aliquots and files to existing runs.
Run Tree
Once you select to execute a workflow, caNanoLab allows you to create a new assay run or to search for an existing run. When you create a new Assay Run, caNanoLab displays the selected Run (Run Tree) in the left hand panel. This facilitates easy navigation within the Run. When you search Runs, described in Searching Runs, search results display in table format. From the table, you can choose a Run to view in detail, while a Run Tree displays in the left side panel.
Upon completing either of these two task options under Execute Workflow (Executing a caNanoLab Workflow), the Assay Run displays in the Run Tree in the left-hand pane of the caNanoLab browser. The workflow displays pre-populated data. There are four assay types in the implemented workflow--Prescreening, Physical, In Vitro, and In Vivo and PCC. Only two assay types, Prescreening and In Vitro, are populated with data in caNanoLab 1.2. Each of the items listed under the assay types are assays.

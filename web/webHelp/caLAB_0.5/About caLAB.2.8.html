Creating a Sample
In a laboratory setting, when new sample comes in from an outside source, you would log it in and store it in a repository. For using it, you would dissolve or dilute the sample, then use it to create aliquots for use at the bench following customized protocols.
In caLAB, you “create” a sample, by recording a physical description of the source sam­ple and its origin. Then you “create” associated aliquots by describing the process by which the source sample is dissolved and/or diluted and otherwise prepared for bench use. You can also view associated protocols that have been previously implemented into caLAB.
Before an Assay Run can be created in caLAB, it is necessary to create one or more samples, then create aliquots of those samples.
 
Note: Protocols must be submitted separately to the caLAB team for incorporation into the caLAB system.
To create a sample, follow these steps:
1.
Click Administration > Manage Samples. This opens the Create Sample form.
2.
Enter or select the appropriate information about the sample you are creating in the text entry fields (described in the following table). Lists associated with drop-down menus are predefined in caLAB. Items that have an asterisk are required fields.
Create Sample Text Fields
Description
Sample ID Prefix*
Represents the fully qualified name of the Sample. This field is pre-determined by the system as the sample is created, however, the field can also be edited. Exam­ple: NCL-8. (NCL is the prefix for the NCL lab.) For more information, see Aliquot Naming Conventions.
Sample Type
Select from the drop-down menu the name of the Sam­ple Type to which the current sample belongs. Exam­ple: Dendrimer, Liposome.
SOP
Select the Standard Operating Procedure type that was used by the lab to generate the sample. You can leave the text field blank, or select between Original or Test­ing.
Note: SOPs can be submitted to caLAB for inclusion in the software as Protocols.
Description
Enter additional comments relevant to the sample cre­ation process.
Source
Enter the name of the originating source for this sam­ple.
Source ID
Enter the unique ID that was assigned by the organiza­tion where the sample originated.
Date Received
Click the calendar to locate the date on which the sam­ple was received by the lab from the source.
Solubility
Enter the value representing how much of a given sub­stance will dissolve in a particular solvent.
Lot ID*
Enter the identifier of the lot such as the lot number for this Sample. This is required for NCL, but “N/A” also permissible.
Lot Description
Enter a description about the lot to which this Sample belongs.
Number of Containers* (Update Containers button)
Enter the number of containers in which the Sample is stored. The container information would dynamically grow based on the Container Number field. (For NCL, “1” is the only permissible value in caLAB 0.5.)
 
Note: If you enter a number great than “1”, complete all of the additional information for the Sample, then return to click the Update Containers button. That way, the information you enter becomes part of your container template and you don’t have to re-enter it again. This is particularly helpful when you are submitting a large number of sample containers. See Step #3.
General Comments
Enter additional information about the sample creation.
Container (Template Container)
Container Type*
Select the Container Type in which the sample is stored in the pre-populated drop-down list. Options: Tube, Vial or Other.
Other
If Other [Container Type] is selected, you must enter the Container Type that is used to store the sample.
Quantity
In the first text box, enter the weight of the sample that is stored in this container. In the drop down list, select the unit used.
Concentration
In the first text box, enter the value of the Concentra­tion of the sample in the current container. In the drop down list, select its associated measurement unit.
 
Note: If Concentration is entered, Diluents/Solvents must also be entered and vice versa.
Volume
In the first text box, enter the value of the Volume in each container in the first text box. In the drop down list, select its associated measurement unit.
Diluents/Solvent
Diluents: Name the liquid used to dilute the liquid sam­ple.
Solvent: Name the solvent used dissolve the solid sam­ple.
 
Note: If Diluents/Solvents is entered, Concentration must also be entered and vice versa.
Safety Precautions
Enter safety precautions that are worth noting.
Example: The Sample is Toxic.
Storage Conditions
Enter the temperature and other variables required to adequately store the sample in the container.
Storage Location
Enter the location in which this container is stored. Example: Freezer1
Comments
Enter additional comments that are associated with this particular sample container.
3.
After you click the Update Container button, the number of containers you entered is reflected by the number of forms that now display in the viewer. Mod­ify each form appropriately to customize the description for each container.)
4.
Click the Submit button after defining the sample(s) appropriately.
After you have clicked the Submit button, if you have failed to enter appropriate information for particular fields, such as the solvent when concentration is entered or the container volume unit, you will receive an error message.You must correct the error before your submission is accepted.
The detailed information you entered displays in table format. A separate table dis­plays for each container you described in your entry.
Samples are used to create aliquots that become part of Assay Runs. For more infor­mation see Creating Aliquots.

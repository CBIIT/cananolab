caNanoLab Functionality Overview
After you have logged into caNanoLab, the application opens automatically to a Welcome page that displays the caNanoLab functions flowchart, represented in the figure in this topic.
Once a user logs in to caNanoLab, with assigned privileges, a user can create samples and aliquots in the Inventory feature. Once a samples is created, with assigned privileges, a user can execute an assay within the laboratory Workflow by creating an assay run, specifying aliquots used within the run, and uploading associated input (sample sheet, instrument configuration) or output (assay result) files. The laboratory workflow acts as a repository for multiple experiment files for each assay execution (run).
Workflows in caNanoLab must be incorporated into the system by the caNanoLab team; they become workflow templates. caNanoLab 1.2 has implemented one workflow, a reference workflow contributed by the Nanotechnology Characterization Laboratory (NCL). Researchers with assigned permissions can use the workflow template to load assay input files, assign aliquots, and load output files (assay results).
After samples are accessioned into caNanoLab, with assigned privileges, a user can Annotate a nanoparticle with functions and characterizations and submit specific assay result files and reports. During particle annotation, the user can submit protocol associations with the nanoparticle and resubmit report metadata. With assigned permissions, annotations can be published to multiple user groups such as the public, CCNE researcher, etc. The annotation feature facilitates information sharing of characterizations and assay results outcomes
For more information about caNanoLab privileges, see Permissions and Visibility.
